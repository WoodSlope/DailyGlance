# DailyGlance

DailyGlance 是一个面向 A 股大盘与自选股观察的网页量化终端，用来集中查看指数环境、个股走势、技术指标和辅助交易结论。

> 仅用于观察、研究和复盘，不构成投资建议。

## 功能

- 大盘：上证指数、创业板指、中证 1000、科创 50
- 自选：搜索股票并加入本地自选
- 图表：K 线、成交量、MACD、KDJ
- 周期：日线 / 周线
- 区间：90 天 / 180 天 / 360 天
- 均线：MA5、MA10、MA20、MA30、MA60、MA120、MA250
- 策略：稳健趋势、波段抄底、突破追涨、综合全能
- 输出：交易结论、建议仓位、防守位、压力区、三条关键依据
- 工具：手动更新、回测、清缓存、设置、帮助、性能诊断

## 页面结构

- 左侧：大盘列表 / 自选股列表
- 中间：主图和指标图
- 右侧：价格、结论、依据、风险提示

## 结论结构

- 第 1 条：大盘定基调
- 第 2 条：指数动能 / 个股信号
- 第 3 条：风控 / 防守

结论卡优先展示最终交易动作，依据只保留必要信息。

## 策略决策与 B/S 标记

策略系统区分技术信号和主图交易标记：

- `B1/B2/...`、`L1/L2/...`、`W1/W2/...` 是技术信号
- 主图 `B/S` 是最终仓位从空仓到持仓、从持仓到空仓的关键转折
- 后续新增常用信号时，优先保证第一次买点和第一次卖点不被误改

详细规则见 [STRATEGY_DECISION_RULES.md](./STRATEGY_DECISION_RULES.md)。

策略打磨流程、候选信号复用方法和本地验证规范见 [STRATEGY_REUSABLE_METHODS.md](./STRATEGY_REUSABLE_METHODS.md)。

## 数据更新规则

- 首屏：先读本地缓存，再做后台刷新
- 侧边栏价格：盘中约每 30 秒批量刷新一次（1 次请求拿全部侧边栏标的实时价格）
  - 大盘页：4 个指数同时刷新
  - 个股页：全部自选股同时刷新
- 当前正在看的大盘或个股：盘中约每 30 秒做一次完整同步（增量历史 + 实时价格 + 图表重绘）
- 侧边栏历史数据：盘中约每 90 秒做一次受控并发同步（并发数 3）
  - 大盘页：4 个指数全部同步
  - 个股页：全部自选股同步
- 切到某只个股后：约 2 秒后会补刷其它自选股
- 手动更新：同一标的 60 秒内限制频繁点击
- 历史数据刷新有冷却：15 秒成功冷却、8 秒尝试冷却
- 分红 / 复权保护：最后一日价格偏差超过 2% 时，不直接增量拼接，改走整段覆盖
- 后台标签页：暂停所有自动刷新，切回前台时若在交易时段立即刷新一次
- 切换标的防抖：快速连续点击切换时只执行最后一次，避免浪费网络请求
- 手动更新冷却：确认弹窗通过后才记录冷却时间，取消不计时
- 手动更新智能确认：数据需要更新时直接同步并 toast 提示，数据已是最新时才弹确认框问是否强制同步
- 移除自选股：改为撤销式，移除后显示 toast「撤销」按钮，3 秒内可恢复，无需确认弹窗
- 刷新视觉反馈：侧边栏价格更新时有脉冲动画，主图区底部显示最近刷新时间
- 错误静默提示：侧边栏连续 2 次刷新失败时 toast 提示，不打断操作
- 历史浏览徽章：查看历史日期时图表区顶部显示「历史浏览 · 日期」徽章，切回最新时自动消失

## 缓存与一致性

- 本地缓存使用浏览器 IndexedDB
- 静态资源通过 `?v=...` 版本参数加载
- 如果本地和 GitHub Pages 的样式、图表或结论不一致，优先检查：
  - 浏览器强制刷新
  - 页面右上角清除缓存
  - `index.html` 中静态资源版本号是否已更新

## 交互优化

- **撤销式移除**：点自选股的 × 不弹确认，直接移除并显示 3 秒撤销 toast，误删可一键恢复
- **智能更新确认**：点「更新数据」时自动判断是否需要更新——需要更新直接执行（toast 提示进度），已是最新才问是否强制同步
- **非阻塞通知**：用 toast 替代弹窗处理非关键信息（移除提示、刷新失败、更新进度），不打断当前操作
- **刷新感知**：侧边栏价格更新时有脉冲动画，主图下方显示最近刷新时间，用户能直观感知数据在更新
- **冻结状态徽章**：浏览历史日期时图表顶部出现「历史浏览」徽章标注当前日期，切回最新时自动消失
- **切换防抖**：快速连点切换标的时只执行最后一次，避免浪费请求

## 设计优化

- **Web 字体**：加载 Inter（UI 文本）和 JetBrains Mono（数字等宽），量化终端数据列对齐
- **键盘焦点**：所有交互元素恢复 `:focus-visible` 样式（2px 蓝色 outline），键盘导航可见
- **缩放自由**：移除 viewport 缩放限制，视力障碍用户可自由放大页面
- **Segmented Control**：激活态从背景色挖洞改为品牌色微染（蓝色半透明底 + 蓝色文字）
- **触控目标**：图标按钮和分段按钮增大 padding，移动端进一步放大，减少误触
- **对比度修复**：evidence-source 透明度从 .68 提升至 .85，10px 小字达到 WCAG AA 4.5:1
- **过渡变量**：统一 5 档过渡时间变量（instant/fast/normal/slow/bounce），消除硬编码时长
- **间距模数**：9 级间距 Token（4/8/12/16/20/24/32/40/48px），非模数值统一 snap 到最近档，消除 11 种散乱间距值
- **字号模数**：10 级字号 Token（10-32px），action-name 从 22px 降至 20px 减少层级跳跃
- **面板密度**：Evidence Grid 从双列改为单列堆叠，标签与来源横向排列，350px 面板下信息不再截断
- **中间断点**：新增 900px 响应式断点，768-1024px 区间导航 200px + 信息 280px + 缩小标题字号，消除布局盲区

## 性能优化

常规使用（鼠标悬停图表、侧边栏 30s 刷新）中的卡顿主要来自 4 块画布全量重绘和侧边栏 DOM 全量重建。当前版本已做以下优化：

- **悬停零重绘**：十字线改为 CSS 绝对定位 div 叠加层，鼠标移动时只移动 div（transform: translateX），4 块画布完全不重绘。悬停帧耗时从 20-60ms 降至 <1ms
- **点击不锁定**：K 线点击不再进入冻结历史浏览，历史查看保留上一天 / 下一天 / 最新、键盘方向键和主图拖拽
- **侧边栏局部刷新**：30s 价格刷新改为只更新 `.lprice` / `.lchange` 文本，不再 `innerHTML` 全量重建。搜索输入框不再被重建，输入焦点不丢失
- **定时器错峰**：两个 30s 定时器（侧边栏价格刷新 / 当前标的完整同步）启动时间错开 15s，避免同时开火造成卡顿脉冲
- **Snapshot 裁剪**：`computeWatchlistDecisionSnapshot` 只计算最后 80 根 K 线（而非全量 ~1000 根），切策略时主线程阻塞从 ~100ms 降至 ~10ms
- **冻结徽章去模糊**：`.freeze-badge` 移除 `backdrop-filter: blur(8px)`，历史浏览翻日期时 GPU 不再重新合成模糊层
